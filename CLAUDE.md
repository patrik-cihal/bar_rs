# Bar Rust - Beyond All Reason MVP

A 3D RTS game inspired by Beyond All Reason, built with Bevy 0.18.

## Build & Run

```bash
cargo build      # compile
cargo run        # run the game

# Regenerate all 3D models from BAR source files (12 models √ó 2 colors = 24 GLBs):
pip install pygltflib Pillow numpy
python3 tools/convert_s3o.py
```

## Architecture

Code is split into modules under `src/`:
- `main.rs` ‚Äî App setup, system registration, GPU workarounds
- `types.rs` ‚Äî Constants, `UnitStats`/`BuildingStats` data tables, components, resources, enums, coordinate helpers (`game_pos`, `game_xy`)
- `spawning.rs` ‚Äî `spawn_unit()`, `spawn_building_entity()` functions
- `setup.rs` ‚Äî Startup systems: camera, lighting, map, HUD, model loading
- `systems/` ‚Äî Update systems, split into submodules:
  - `input.rs` ‚Äî cursor, camera, selection, build mode, building placement, factory queue, D-Gun
  - `movement.rs` ‚Äî unit movement, building construction
  - `combat.rs` ‚Äî combat system, projectile system
  - `economy.rs` ‚Äî resource production, factory production, reclaim, wreckage decay
  - `environment.rs` ‚Äî collision, terrain follow, fog of war
  - `visuals.rs` ‚Äî death explosion, health bars, selection indicators, build ghost, HUD, minimap, win/lose
  - `animation.rs` ‚Äî commander walk, artillery walk, vehicle turret aim, building animations (extractor spin, radar dish, LLT turret, factory cagelight), unit facing

Tools:
- `tools/convert_s3o.py` ‚Äî Batch downloads BAR S3O models + shared texture, converts all to glTF (.glb) with team color applied (blue + red variants for each)
- `tools/view_model.html` ‚Äî Three.js model viewer for debugging (open in browser, needs local file server)

Assets:
- `assets/models/{name}_{blue,red}.glb` ‚Äî 3D models for all units and buildings (24 files total, generated by convert_s3o.py)
- Models: armcom (Commander), armpeep (Scout), armflash (Raider), armstump (Tank), armbull (Assault), armham (Artillery), armmex (Metal Extractor), armsolar (Solar), armlab (Factory), armllt (LLT), armdrag (Dragon's Teeth), armrad (Radar Tower)

### 3D Rendering

- **Camera**: `Camera3d` with orthographic projection, ~55¬∞ RTS-style angle
- **Coordinate system**: Game (x, y) maps to world (x, layer, -y) via `game_pos()` / `game_xy()` helpers in types.rs
- **All units and buildings**: Real BAR models loaded as `SceneRoot` from glTF, with team-colored textures (blue/red)
- **Commander**: Has procedural walk animation via `CommanderWalkAnim`
- **Non-commander units**: Face movement/attack direction via `unit_facing_system`
- **Fallback**: Cylinder (units) / Cuboid (buildings) if model not found
- **Terrain**: Procedural heightmap mesh (101x101 grid) with multi-octave noise, vertex-colored (green/brown/grey by height), flattened near spawn areas and map edges
- **Lighting**: Two `DirectionalLight`s (sun with shadows + fill)
- **GPU fix**: `NoIndirectDrawing` on camera + disabled GPU preprocessing (required for Intel iGPUs)

### ECS Structure

**Components:**
- `Unit` ‚Äî HP, speed, attack stats, cooldown timer, min_attack_range
- `Commander`, `Tank`, `Scout`, `Raider`, `Assault`, `Artillery` ‚Äî unit type markers
- `PlayerOwned`, `EnemyOwned` ‚Äî team markers
- `Building` ‚Äî build progress, completion state, building_type
- `MetalExtractor`, `SolarCollector`, `Factory`, `LightLaserTower`, `DragonTeeth`, `RadarTower` ‚Äî building type markers
- `Factory` ‚Äî has `queue: Vec<UnitType>`, `produce_timer`, `current_build_time`
- `MetalSpot`, `MapFeature` ‚Äî map features (metal spots, reclaimable rocks/trees)
- `Wreckage` ‚Äî spawned on unit death, reclaimable for metal, decays over time
- `Selected`, `MoveTarget(Vec2)`, `AttackTarget(Entity)`, `ReclaimTarget(Entity)` ‚Äî unit state
- `SightRange(f32)`, `RadarRangeComp(f32)` ‚Äî vision system
- `Projectile` ‚Äî target, damage, speed, is_dgun flag
- `DeathExplosion` ‚Äî expanding circle visual on commander death
- `CommanderWalkAnim` ‚Äî walk cycle phase and active state for commander animation
- `BipedWalkAnim` ‚Äî walk cycle phase for artillery (armham) biped animation
- `VehicleAnim` ‚Äî marker for vehicle turret aim animation (tank, raider, assault)

**Resources:**
- `GameResources` ‚Äî metal, energy, income rates
- `BuildMode` ‚Äî active build placement state
- `DragSelect` ‚Äî box selection state
- `CursorWorldPos` ‚Äî cached world-space cursor position
- `GameOver` ‚Äî win/lose state
- `DGunMode` ‚Äî D-Gun targeting active
- `ModelLibrary` ‚Äî HashMap of all loaded model Handle<Scene>s (keyed by "{name}_{color}")
- `TerrainHeightmap` ‚Äî Procedural heightmap grid with `height_at(x, y)` for terrain queries

**Systems (Update schedule, three groups):**
1. Chained: cursor ‚Üí camera ‚Üí selection ‚Üí build mode ‚Üí building placement ‚Üí unit commands ‚Üí factory queue ‚Üí dgun ‚Üí movement ‚Üí construction
2. Parallel: combat ‚Üí projectile ‚Üí resource production ‚Üí factory production ‚Üí reclaim ‚Üí wreckage decay ‚Üí collision ‚Üí fog of war ‚Üí terrain follow
3. Parallel: death explosion ‚Üí health bars ‚Üí selection indicators ‚Üí build ghost ‚Üí HUD ‚Üí minimap ‚Üí win/lose
4. Parallel: commander animation ‚Üí biped walk ‚Üí vehicle turret ‚Üí building animation ‚Üí LLT turret ‚Üí unit facing ‚Üí artillery facing

## Bevy 0.18 Specifics

- `MessageReader<T>` instead of `EventReader<T>` (import from `bevy::ecs::message`)
- Query `Projection` enum, not `OrthographicProjection` directly (it's not a Component)
- `OrthographicProjection::default_3d()` for 3D ortho camera
- `WindowResolution::from([1280u32, 720])` ‚Äî no float tuple conversion
- `Children` derefs to `[Entity]`; `.iter()` yields `Entity` (Copy type)
- `.chain()` has a tuple size limit (~12 systems); split into multiple `add_systems` calls
- Use `Without<T>` filters to make overlapping queries disjoint
- `SceneRoot(Handle<Scene>)` to spawn glTF scenes
- `NoIndirectDrawing` from `bevy::render::view` ‚Äî required on Intel iGPUs
- `GpuPreprocessingSupport` from `bevy::render::batching::gpu_preprocessing` ‚Äî set to None for Intel iGPUs

## Game Controls

- **WASD / Arrow keys** ‚Äî pan camera
- **Scroll wheel** ‚Äî zoom
- **Left click** ‚Äî select unit
- **Drag left click** ‚Äî box select
- **Right click** ‚Äî move / attack / reclaim (commander on wreckage/features)
- **1-6** ‚Äî commander build menu: [1] Extractor, [2] Solar, [3] Factory, [4] LLT, [5] Wall, [6] Radar
- **Q/W/E/R/T** ‚Äî factory unit queue: Scout, Raider, Tank, Assault, Artillery (Shift+key for x5)
- **G** ‚Äî D-Gun mode (commander, costs 500 energy, one-shot kill, short range)
- **Esc** ‚Äî cancel build/D-Gun mode

## Key Mechanics

- **Factory Queue**: Player factories produce units from a queue; enemy factories auto-produce tanks
- **Wreckage + Reclaim**: Dead units leave wreckage; commander can reclaim for metal; map has rocks/trees
- **LLT**: Stationary turret, auto-attacks enemies (25 dmg, 250 range)
- **Dragon's Teeth**: Wall that repels units (movement blocker)
- **D-Gun**: Commander fires 9999-damage projectile, doesn't work on commanders
- **Commander Death**: 500 AoE damage in 200 radius, leaves wreckage
- **Fog of War**: Units outside friendly sight range are hidden; radar extends detection
- **Minimap**: Click bottom-left box to move camera

## S3O Model Pipeline

All unit and building models come from the BAR repository. `tools/convert_s3o.py`:
1. Downloads 12 S3O models + shared `Arm_color.dds` texture from BAR GitHub (cached in `tools/.cache/`)
2. Parses S3O binary format (piece hierarchy, vertices, normals, UVs, indices)
3. Applies team color via texture alpha channel (alpha=255 ‚Üí full team color, alpha=0 ‚Üí original)
4. Negates X coordinates (S3O convention), swaps triangle winding to compensate
5. Shrinks vertices 5% toward centroid to reduce Z-fighting at joints
6. Skips emitter/flare/anchor pieces with no useful geometry; commander-specific medal/crown skip
7. Outputs 24 .glb files (12 models √ó blue + red) with embedded PNG textures
